# Personal Internet Memory Bot - GENERAL Design Document

## Summary
The Personal Internet Memory Bot is a Telegram-based personal archive system for individuals to save, organize, and recall digital content (articles, tweets, videos, messages) using AI-powered tagging, summarization, and spaced repetition. Users forward content to the bot, which automatically processes and stores it, then resurfaces relevant items weekly via direct Telegram messages. The bot supports natural language queries to retrieve specific saved items with links and summaries.

## Core Entities
- **User**: Telegram user interacting with the bot (1:N relationship with Saved Items, Collections, and Spaced Repetition Schedules).
- **Saved Item**: Content (text, link, media) forwarded by a user. Attributes: content type, original message data, timestamp, AI summary, tags.
- **Tag**: AI-generated keyword (N:N relationship with Saved Items via a join table).
- **Summary**: AI-generated text (1:1 relationship with Saved Item).
- **Collection**: Logical grouping of Saved Items (1:N relationship with Saved Items). Can be auto-generated by tag similarity or user-defined.
- **Spaced Repetition Schedule**: Algorithmic schedule for resurfacing items (1:N relationship with Saved Items).

## External Dependencies
- **Telegram Bot API**: Message forwarding, inline queries, scheduled notifications.
- **AI Services**: Tagging (e.g., NLP keyword extraction) and summarization (e.g., GPT-3.5/4, BERT-based models).
- **Database**: Store users, saved items, tags, summaries, collections, and spaced repetition intervals.
- **Media Storage**: Optional cloud storage for large media files (if required by Telegram's message limits).

## Full Feature List
- **Content Ingestion**: Accept forwarded messages (text, links, images, videos) from users.
- **AI Tag Generation**: Automatically extract 3-5 relevant tags per saved item using NLP.
- **AI Summary Generation**: Create concise summaries (100-200 words) for text-based content.
- **Collection Auto-Grouping**: Group items into collections based on tag overlaps or user-defined rules.
- **Weekly Spaced Repetition**: Send a curated list of 5-10 items to users weekly, prioritizing older content.
- **Natural Language Search**: Parse queries like "Show me the article about Go architecture from March" and return matching items with links/summaries.
- **Collection Management**: Allow users to rename, delete, or view specific collections.
- **Dashboard View**: Provide a summary of all saved items, collections, and upcoming reminders.
- **Confirmation Notifications**: Send a message confirming content was saved with its summary and tags.
- **Tag Filtering**: Enable users to view all items with a specific tag.

## Non-Goals
- Public sharing of saved content.
- Real-time collaborative features (e.g., shared collections).
- Manual content editing (e.g., modifying tags/summaries).
- Integration with external platforms beyond Telegram.
- Customizable spaced repetition intervals beyond weekly.
